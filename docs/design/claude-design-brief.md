# Claude Design 向けデザインブリーフ: Yaeh Map リデザイン

収益化戦略（GitHub Issue #103）に伴うアプリケーションデザイン刷新のためのブリーフ。Claude Design への指示のベースとして使う。

---

## 1. プロダクト概要とブランド

- **Yaeh Map（ヤエーマップ）**: バイクユーザ専用のツーリング情報共有SNS。https://yaehmap.com
- 名前の由来: 「ヤエー（v(・∀・)yaeh!）」＝すれ違うライダー同士が交わす挨拶のピースサイン。旅の安全を願う文化
- ミッション: **「バイクの事故を減らす」×「バイクをもっと楽しんでほしい」**
- ポジショニング: 映え写真SNS（モトクル）ではなく、**「山間部の道路状況 × 速報性」の実用・安全情報サービス**。ツーリング先を決め、危険を事前に知り、Google マップアプリに目的地を渡すところまでが体験の核
- いいねボタンは「ヤエー」（遊び心はブランド資産として維持する）

### 現状のブランド課題
- 現テーマは **shadcn-vue デフォルトの無彩色（グレースケール oklch）のままでブランドカラー未定義**。ここが今回の最大の伸びしろ
- 旧デザイン（卒業発表資料時代）は青系（Bootstrap ブルー基調）だったが、踏襲義務はない
- ロゴは favicon 程度しかない。ワードマーク＋シンボル（ヘルメット/ピースサイン/地図ピンなどのモチーフ）の検討余地あり

### トーンの方向性（提案）
- 「信頼できる実用ツール」（道路標識・地図の視覚言語）＋「ライダー仲間のフレンドリーさ」（ヤエー文化）
- 参考にすべき視覚文脈: ツーリングマップル、道路標識の色体系（警戒＝黄、規制＝赤 等）、アウトドアナビアプリ

## 2. ターゲットユーザーと利用文脈（デザイン制約として最重要）

| シーン | 環境 | デザイン要件 |
|---|---|---|
| 出発前の計画 | 自宅・落ち着いている | 情報密度高めOK、地図での探索体験 |
| 休憩中の閲覧・投稿 | **屋外・直射日光・グローブの可能性・短時間** | 高コントラスト、大タップターゲット（44px+）、最短投稿フロー、片手・親指操作 |
| 帰宅後の投稿・振り返り | 自宅 | 通常のフォーム体験でOK |

- **スマホファースト**（レスポンシブ）。PWA としてホーム画面追加を促す方針（ネイティブアプリは当面作らない）
- ダークモード対応必須（夜間走行後・トンネル・屋内）。`.dark` クラス切替で実装済み
- ナビゲーション自体は作らない: 「Google マップアプリへ目的地/経由地を渡す」ボタンが体験の出口

## 3. 技術制約（成果物をそのまま実装に落とせる形式）

- スタック: **Vue 3 + TypeScript + Tailwind CSS v4 + shadcn-vue (reka-ui) + lucide アイコン**
- デザイントークンは `frontend/src/styles/app.css` に **shadcn-vue 規約の CSS 変数（oklch）** で定義: `--primary` `--secondary` `--accent` `--destructive` `--card` `--muted` `--border` `--ring` `--radius`（0.625rem）ほか。`:root`（ライト）と `.dark` の両方
- **希望する出力形式**: ① shadcn-vue 変数名にマップした oklch カラーパレット（ライト/ダーク両方）、② Tailwind ユーティリティクラスベースのコンポーネント例、③ アイコンは lucide のアイコン名指定
- 導入済み UI プリミティブ: alert / avatar / badge / button / card / carousel / checkbox / collapsible / input / label / pagination / popover / radio-group / select / sheet / table / textarea（+ native-select スタイル）
- 追加が見込まれるプリミティブ: tabs, dialog / drawer（投稿フロー・ボトムシート）, switch, toast, skeleton, bottom-navigation（カスタム）
- 地図は Google Maps JavaScript API。**地図タイル自体の色はいじれない前提**で、地図の上に載る UI（カスタムピン、バッジ、ボトムシート、カード）をデザインする

## 4. 現在の画面一覧（14画面）と主要コンポーネント

- 画面: トップ（LP的位置づけ）/ ログイン / 新規登録 / パスワード忘れ・再設定 / 投稿一覧（タイムライン+Ransack検索+ページネーション）/ 投稿作成・詳細・編集 / プロフィール表示・編集 / 認証情報変更 / 管理者（アカウント一覧）/ 404
- 主要コンポーネント: NavBar（ハンバーガーメニュー）、PostCard（静的地図サムネ+タイトル+投稿者+タグ+ヤエーボタン）、PostForm、MapPicker（位置選択）、PostTimeline、SearchDropdown、FavoriteButton、PaginationBar、PortraitIcon、FlashMessages
- 投稿の属性: タイトル / 地域・都道府県 / ジャンル（現在は「オススメ・駐輪場・注意」の3種）/ 場所名 / 説明 / 写真（最大4枚）/ 緯度経度

## 5. 収益化戦略で必要になるデザイン変更・新規要素

### Phase 0: 信頼の土台（Issue #104）
1. **位置精度セレクタ**（投稿フォーム）: 「正確 / おおまか（約1km）/ 位置なし」の3択 UI。プライバシーを守っている安心感を伝えるマイクロコピーとアイコン
2. **位置なし投稿のカードバリアント**: 地図サムネイルなしでも間延びしないレイアウト
3. **通報フロー**: 投稿メニュー → 理由選択 → 送信完了。管理画面側の通報キュー（テーブル+対応状況）
4. 利用規約 / プライバシーポリシー / 特商法表記の文書ページテンプレート

### Phase 1: 差別化の核（Issue #105）— ここが最優先のデザイン対象
5. **道路状況ジャンルの視覚言語（最重要）**: 「オススメ / 駐輪場」に加え、「通行止め / 凍結 / 砂利・落石 / 工事 / 冬季閉鎖」等の**アイコン＋色の体系**。道路標識メタファー。地図ピン・フィードバッジ・投稿フォームの3か所で一貫させる
6. **鮮度・有効期限の表現**: 「あと3時間」「今日中」「期限切れ（アーカイブ）」のバッジとカードの経年表現（新しい=鮮明、古い=退色）。速報性が差別化なので時間表示は一級市民
7. **地図ファースト画面**: フルスクリーン地図＋ジャンル別ピン＋下からのボトムシート（カードリスト）。現行タイムラインとのタブ/トグル切替
8. **ボトムナビゲーション**: 地図 / タイムライン / 投稿(+) / 通知 / マイページ。親指到達圏
9. **投稿フロー再設計**: カメラ起動 → 写真 → 位置（精度選択）→ ジャンル → 一言、の最短ステップ（休憩中に1分で投稿完了）
10. **PWA インストール誘導**: iOS の「ホーム画面に追加」手順ガイド UI
11. **OGP シェアカード**（1200×630）: 静的地図＋ジャンルバッジ＋タイトル＋ブランド

### Phase 2: 収益レール
12. **料金ページ**: 無料 / プレミアム（月300〜500円想定）比較表。訴求軸は「広告非表示・プラン無制限・GPX」。**安全情報は全員無料であることを明示**（ブランドの信頼に直結）
13. **文脈内アップセル**: プラン4個目作成時などに出す非破壊的なアップグレード導線（閲覧は絶対に止めない）
14. **ツーリングプラン**: 作成（経由地リスト＋地図）、詳細（共有 / GPX / 「Google マップで開く」ボタン）、一覧
15. **広告スロット**: フィード内ネイティブ広告カード（「広告」ラベル必須）、詳細ページ下部バナー。コンテンツと区別しつつ体験を壊さないデザイン
16. **空状態（Empty State）デザイン**: 初期はコンテンツが少ないSNSなので、ゼロデータ画面が投稿を促す導線になるよう重視

### Phase 3: スポンサー
17. **スポンサーピン＋店舗ページ**: 地図上で目立つが「PR」表記必須（ステマ規制対応）。バイク用品店・整備工場・ライダーズカフェ想定

## 6. デザイン原則（収益化との整合ルール）

1. **安全情報（注意系ジャンル）は最も視認性が高く、無料で、広告と絶対に混同されない**
2. 広告・スポンサーには明確なラベル（「広告」「PR」）
3. プレミアム誘導は閲覧・安全情報の邪魔をしない
4. 鮮度（時間）の表現をどの画面でも一貫させる
5. ライト/ダーク両テーマで上記すべてが成立すること

## 7. Claude Design への依頼順序(提案)

1. **ブランド基盤**: カラーパレット（shadcn-vue 変数への oklch マッピング、ライト/ダーク）、タイポグラフィ、ジャンルアイコン＋色体系、ヤエーボタンの表現、ロゴ方向性
2. **コア画面**: 地図ファースト画面（ピン+ボトムシート）、PostCard 新バリアント（位置なし/期限つき/広告）、投稿フロー、ボトムナビ
3. **収益系画面**: 料金ページ、アップセル、プラン作成・詳細
4. 成果物はコンポーネントカード単位で claude.ai/design のデザインシステムプロジェクトに保存し、DesignSync で同期できる形にする
